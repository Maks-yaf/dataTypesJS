Data types

A value in JavaScript always refers to a specific data type. For example, it could be a string or a number.

There are eight main data types in JavaScript. In this chapter we will look at them in general, and in subsequent chapters we will talk in more detail about each.

A variable in JavaScript can contain any data. At one moment there may be a string, and at another - a number:

              // Won't be a mistake
              let message = "hello";
              message = 123456;

Programming languages that allow this are called “dynamically typed”. This means that there are data types, but the variables are not bound to any of them.

Number
              let n = 123;
              n = 12.345;

The numeric data type (number) represents both integer and floating-point values.

There are many operations for numbers, such as multiplication *, division /, addition +, subtraction - and so on.

In addition to ordinary numbers, there are so-called “special numeric values” that belong to this data type: Infinity, -Infinityand or NaN.

  • Infinity represents the mathematical infinity ∞. This is a special value that is greater than any number.

String

The string (string) in JavaScript must be enclosed in quotation marks.

              let str = "Hello";
              let str2 = 'Single quotes are fine too';
              let phrase = `Backticks allow variables to be embedded ${str}`;

There are three types of quotes in JavaScript.

  1. Double quotes: "Hello".
  2. Single quotes: 'Hello'.
  3. Backticks: `Hello`.

Double or single quotes are "single" quotes; there is no difference between them in JavaScript.

Backticks have extended functionality. They allow us to embed expressions in a string by enclosing them in ${…}. For example:

            let name = "Maks";
            alert( `Hello, ${name}!` ); // Hello, Maks!
            alert( `Result: ${1 + 2}` ); // Result: 3

The expression inside ${…} is evaluated and its result becomes part of the string. We can put whatever we want there: a variable name, or an expression 1 + 2, or something more complex.

Boolean (logical) type

The Boolean type (boolean) can only take two values: true and false.

This type is typically used to store yes/no values: true it means “yes, right”, and false means “no, not right.”

For example:

             let nameFieldChecked = true; // yes, is checked
             let ageFieldChecked = false; // no, is not checked

Boolean values can also be the result of comparisons:

            let isGreater = 4 > 1;
            alert( isGreater ); // true (the result of the comparison will be "yes")
Null

The special value null is not one of the types described above.

It forms a separate type that contains only the value null:

In JavaScript, null it is not a "reference to a non-existent object" or a "null pointer" as in some other languages.

It is simply a special value that represents "nothing", "empty" or "unknown value".

Undefined

The special meaning undefined also stands out. It forms a type from itself in the same way as null.

It means "no value was assigned."

If a variable is declared, but no value is assigned to it, then its value will be undefined:

              let age;
              alert(age); // will show "undefined"
Object and symbols

The type object is special.

All other types are called "primitive" because their values can only be simple values (be it a string, or a number, or something else). Objects store collections of data or more complex structures.

A type symbol(character) is used to create unique identifiers in objects. We mention it here for the sake of completeness; we will study this type after objects.

Typeof operator

The operator typeof returns the type of the argument. This is useful when we want to treat values of different types differently or just want to do some checking.

It has two syntactic forms:

              typeof 5; // Output "number"
              typeof(5); // Also output "number"

If an expression is passed, then it must be enclosed in parentheses, because typeof has higher precedence than binary operators:

              typeof 50 + " Apartments"; // Output "number Apartments"
              typeof (50 + " Apartments"); // Output "string"

In other words, the parentheses are needed to determine the type of the value that results from executing the expression within them.

The call typeof x returns a string with the type name

              typeof undefined // "undefined"
              typeof 0 // "number"
              typeof 10n // "bigint"
              typeof true // "boolean"
              typeof "foo" // "string"
              typeof Symbol("id") // "symbol"
              typeof Math // "object"  (1)
              typeof null // "object"  (2)
              typeof alert // "function"  (3)

The last three lines need clarification:

  1. Math is a built-in object that provides mathematical operations and constants. Here it serves only as an example object.
  2. The result of the call typeof null is "object". This is an officially recognized bug in typeof, dating back to the inception of JavaScript and kept for compatibility. Of course null it is not an object. This is a special value with a separate type.
  3. The call typeof alert returns "function" because alert it is a function. We'll explore functions in the following chapters, where we'll also see that JavaScript doesn't have a special function type. Functions are of the object type. But typeof it processes them in a special way, returning "function". This also happened from the creation of JavaScript. This is formally incorrect, but can be convenient in practice.
Total

There are 8 main data types in JavaScript.

  • Seven of these are called "primitive" data types:
    • number for any numbers: integers or floating point numbers; integer values are limited to range .±(253-1)
    • bigint for integers of arbitrary length.
    • string for strings. A string can contain zero or more characters; there is no separate character type.
    • boolean For true/ false.
    • null for unknown values – a separate type that has one value null.
    • undefined for unassigned values, a separate type that has one value undefined.
    • symbol for unique identifiers.
  • And one is not “primitive” and stands apart:
    • object for more complex data structures.

The operator typeof allows us to see what type of data is stored in the variable.

  • It has two forms: typeof xor typeof(x).
  • Returns a string with the name of the type. For example, "string".
  • For null returns "object" is a bug in the language; it's not actually an object.
